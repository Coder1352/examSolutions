Question 2

(a) Explain why data is stored in computers in binary format.

Data is stored on different components in computers such as the CPU, RAM and secondary storage, as well as transistors and logic gates. These components contain circuits. Computers use and store information in binary format, which is a system of representing information using only two digits: 0 and 1. These values are physically represented within a computer, when voltage flows through a circuit or not. In many computers a voltage of 3.3 or 5 volts flowing through a circuit represents a 1. 0 volts flowing through the circuit represents a 0. The use of binary information allows computers to rely on Boolean logic, which is an important part of computer programming. In addition, all computers have a finite storage space and they aren't suitable for storing large quantities of continuous data. So, it's necessary to store information in a discrete or binary form.

Question 7

(a) Differentiate between HTTP and HTTPS

Both HTTP and HTTPS are communication protocols, which are a set of rules for transmitting data across the internet. The client-server model uses several different protocols. HTTP and HTTPS are two layers of protocols in the client-server model.
HTTP stands for hyper text transfer protocol, which is a request-response protocol. It allows a client to issue a request for a resource (such as a web page) and the server to respond.
HTTPS stands for hyper text transfer protocol secure. It is a secure version of HTTP. When using HTTPS, data transmitted between the client and server is encrypted. This prevents anyone who intercepts the data to understand the data. HTTPS has become the standard on the internet in recent years, particularly for online payments.

(b)

Explain how the TCP communication protocol provides reliable communication between computers.

Transmission control protocol operates at the transport layer, ensuring that binary information is transported reliably between computers. The infomration is broken up into packets for efficient routing. TCP keeps track of each of these packets, ensuring that they remain free of error and arrive at the client in the same order as those tansmission by the server.
